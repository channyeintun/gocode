│ Goal Description                                                                                                                                                                                       │
│                                                                                                                                                                                                        │
│ Design a VS Code Copilot–style subagent flow for `gocode` with three distinct child-agent experiences:                                                                                                 │
│                                                                                                                                                                                                        │
│ 1. Explore — fast, read-only codebase research                                                                                                                                                         │
│ 2. Search — iterative search-focused subagent that returns structured file/snippet references                                                                                                          │
│ 3. Execution — execution-focused subagent that runs terminal tasks and returns compact command summaries                                                                                               │
│                                                                                                                                                                                                        │
│ The implementation should preserve `gocode`'s existing plan-mode review gate, permission model, and artifact workflow while introducing clearer child-agent specialization similar to                  │
│ `vscode-copilot-chat`.                                                                                                                                                                                 │
│                                                                                                                                                                                                        │
│ Current State                                                                                                                                                                                          │
│                                                                                                                                                                                                        │
│ `gocode` already has a strong subagent foundation:                                                                                                                                                     │
│                                                                                                                                                                                                        │
│ - `agent`, `agent_status`, and `agent_stop` tools already exist and are registered in `cmd/gocode/engine.go`.                                                                                          │
│ - Background child-agent lifecycle updates are already emitted over IPC via `background_agent_updated` and rendered in the TUI (`tui/src/hooks/useEvents.ts`,                                          │
│ `tui/src/components/BackgroundAgentsPanel.tsx`).                                                                                                                                                       │
│ - Subagents already run in isolated sessions with filtered tool registries and cloned permission context in `cmd/gocode/subagent_runtime.go`.                                                          │
│ - Two subagent types already exist:  - `explore` → read-only tool list                                                                                                                                 │
│     - `general-purpose` → broader tool list with cloned permissions                                                                                                                                    │
│ - Plan-mode review gates already exist through `save_implementation_plan` and artifact review events.                                                                                                  │
│                                                                                                                                                                                                        │
│ The main gaps versus VS Code Copilot’s flow are:                                                                                                                                                       │
│                                                                                                                                                                                                        │
│ - no dedicated search or execution subagent types                                                                                                                                                      │
│ - no specialized prompt contracts for those modes                                                                                                                                                      │
│ - no structured return formats tuned for search references or execution summaries                                                                                                                      │
│ - no first-class tool descriptions that steer the parent agent toward the right child-agent mode for a task                                                                                            │
│ - no dedicated UX language in the TUI/README explaining the three-mode subagent workflow                                                                                                               │

│ User Review Required                                                                                                                                                                                   │
│                                                                                                                                                                                                        │
│ Please review these key design decisions before implementation:                                                                                                                                        │
│                                                                                                                                                                                                        │
│ 1. Should `search` be workspace-only?  - Recommended: yes; exclude `web_search` and `web_fetch` to keep it analogous to Copilot’s search subagent.                                                     │
│ 2. Should `execution` be terminal-only, or also allow minimal file inspection?  - Recommended: allow `bash` plus background-command follow-up tools and possibly read-only inspection tools like       │
│ `file_read`/`list_dir` if needed for context.                                                                                                                                                          │
│ 3. Should `execution` remain non-writing by default?  - Recommended: yes. Keep it execution-focused rather than turning it into another general-purpose coding subagent.                               │
│ 4. What should the final-answer contract be for `search`?  - Recommended: path-and-line-range references first, with optional runtime hydration later.                                                 │
│ 5. Should `explore` remain the default `subagent_type` for backward compatibility?  - Recommended: yes.                                                                                                │
